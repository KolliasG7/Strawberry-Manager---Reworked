# Building the iOS App - Important Information

## Why No IPA Yet?

The Swift iOS source code is **complete and production-ready**, but building an actual IPA requires creating an **Xcode project file** first. This is a one-time setup that must be done locally with Xcode.

## Why Can't GitHub Actions Build It Automatically?

iOS apps require:
1. **Xcode project file** (.xcodeproj) - GUI-generated configuration
2. **Code signing certificates** - Apple Developer account required
3. **Provisioning profiles** - Device/distribution certificates
4. **macOS with Xcode** - Cannot be fully automated without setup

GitHub Actions **will build automatically** once you commit the `.xcodeproj` file after initial setup.

## 🎯 Two Options to Build

### Option A: Create Xcode Project (Recommended for App Store)

**Time Required:** 15-20 minutes (one-time setup)

**Steps:**

1. **Open Xcode:**
   ```bash
   open -a Xcode
   ```

2. **Create New Project:**
   - File → New → Project
   - Select "iOS" → "App"
   - Click "Next"

3. **Configure Project:**
   - Product Name: `StrawberryManager`
   - Team: Select your Apple Developer team
   - Organization Identifier: `com.yourname`
   - Interface: **SwiftUI**
   - Language: **Swift**
   - Storage: None
   - Click "Next" and choose location

4. **Replace Default Files:**
   ```bash
   # Delete default ContentView.swift and StrawberryManagerApp.swift from Xcode
   
   # Drag and drop these folders from Finder into Xcode project:
   - StrawberryManager-iOS/Models/
   - StrawberryManager-iOS/ViewModels/
   - StrawberryManager-iOS/Views/
   - StrawberryManager-iOS/Services/
   - StrawberryManager-iOS/Utilities/
   - StrawberryManager-iOS/StrawberryManagerApp.swift
   
   # When prompted:
   ✅ "Copy items if needed"
   ✅ "Create groups"
   ✅ Add to target: StrawberryManager
   ```

5. **Configure Build Settings:**
   - Select project in navigator
   - General tab:
     * Minimum Deployments: iOS 17.0
   - Signing & Capabilities:
     * Team: Your Apple Developer team
     * ✅ Automatically manage signing

6. **Add App Icon:**
   - Open Assets.xcassets
   - Select AppIcon
   - Drag `assets/logo.png` into 1024×1024 slot

7. **Configure Info.plist:**
   ```xml
   <key>NSLocalNetworkUsageDescription</key>
   <string>Connect to PlayStation 4 on local network</string>
   <key>NSAppTransportSecurity</key>
   <dict>
       <key>NSAllowsArbitraryLoads</key>
       <true/>
   </dict>
   ```

8. **Build:**
   ```bash
   # In Xcode:
   Product → Build (⌘B)
   
   # Run on simulator:
   Product → Run (⌘R)
   
   # Archive for distribution:
   Product → Archive
   ```

9. **Commit Xcode Project:**
   ```bash
   git add StrawberryManager.xcodeproj
   git commit -m "Add Xcode project configuration"
   git push origin main
   
   # Now GitHub Actions will build automatically!
   ```

---

### Option B: Build with Swift Package Manager (No GUI)

**Time Required:** 5 minutes

**Limitations:**
- Creates a library, not a standalone app
- Cannot produce IPA file
- Cannot deploy to App Store
- Good for testing/validation only

**Steps:**

```bash
cd StrawberryManager-iOS

# Build the package
swift build

# Run tests
swift test

# This validates code compiles but doesn't create an app
```

## 🤔 Which Option Should You Choose?

### Choose Option A (Xcode Project) if:
- ✅ You want to distribute via TestFlight or App Store
- ✅ You want to run on physical iPhone
- ✅ You want GitHub Actions to build automatically
- ✅ You want a complete iOS app

### Choose Option B (Swift Package) if:
- ✅ You just want to verify code compiles
- ✅ You're doing code review
- ✅ You don't need an actual app yet

## 🚀 What Happens After Xcode Setup?

Once you commit the `.xcodeproj` file:

1. **GitHub Actions becomes fully automated:**
   ```yaml
   # .github/workflows/ios-build.yml will:
   - Detect Xcode project ✅
   - Build for iOS Simulator automatically
   - Run tests
   - Archive build logs
   ```

2. **Builds work in CI/CD:**
   - Every push triggers automatic build
   - Download build logs from Actions artifacts
   - See build status in PR checks

3. **Manual workflow runs enabled:**
   - Actions tab → iOS Build → "Run workflow" button
   - Build on-demand anytime

## 💡 Why This Design?

The Swift iOS implementation provides **complete source code** that's:
- ✅ Production-ready (3,500+ lines)
- ✅ All features implemented
- ✅ Proper architecture (MVVM + Combine)
- ✅ SwiftUI previews for development

But the `.xcodeproj` file contains **machine-specific configurations**:
- Bundle identifiers (must be unique to you)
- Code signing team (your Apple Developer account)
- Build settings (your preferences)
- Asset catalogs (generated by Xcode)

That's why it requires one-time local setup before automation kicks in.
